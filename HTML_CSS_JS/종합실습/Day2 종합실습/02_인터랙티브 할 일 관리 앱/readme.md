## Doing

이번 과제에서는 할 일 관리 앱을 만들면서 JavaScript의 변수, 배열, 객체, 조건문, 반복문, 배열 메서드, DOM 조작, 이벤트 처리를 실제 기능에 연결해봤습니다. 할 일 하나는 `{ id, text, done }` 형태의 객체로 만들고, 전체 할 일 목록은 배열로 관리했습니다.

할 일을 추가할 때는 입력값의 앞뒤 공백을 제거한 뒤 빈 값이면 추가되지 않도록 막았습니다. 정상 입력인 경우에는 새로운 객체를 만들고 배열에 `push()`로 추가했습니다. 입력 후에는 입력창을 비우고 다시 렌더링하도록 구성했습니다. 추가 버튼뿐만 아니라 입력창에서 Enter 키를 눌러도 form submit 이벤트로 같은 추가 동작이 실행되도록 만들었습니다.

할 일 목록은 JavaScript로 DOM을 생성해 화면에 표시했습니다. `createElement()`로 `li`, `input`, `span`, `button`을 만들고, 각 항목의 id는 `dataset`에 저장했습니다. 완료 처리는 체크박스를 클릭하면 `done` 값을 반대로 바꾸는 방식으로 구현했고, 완료된 항목에는 취소선이 표시되도록 class를 적용했습니다.

삭제 기능은 X 버튼을 클릭했을 때 해당 id가 아닌 항목만 남기도록 `filter()`를 사용했습니다. 전체, 진행중, 완료 필터도 현재 필터 값에 따라 배열을 걸러서 렌더링하는 방식으로 구현했습니다. 화면 하단에는 전체 개수와 완료 개수를 요약해서 보여주도록 만들었습니다.

이벤트 처리는 할 일 항목마다 따로 리스너를 붙이지 않고, 부모 `ul`에 이벤트 리스너를 한 번만 등록했습니다. 클릭된 요소가 체크박스인지 삭제 버튼인지 확인한 뒤 각각 완료 토글 또는 삭제를 처리했습니다. 이를 통해 동적으로 추가되는 항목도 같은 이벤트 처리 흐름 안에서 동작하도록 만들었습니다.

데이터 저장은 `localStorage`를 사용했습니다. 할 일이 추가, 완료, 삭제될 때마다 배열을 JSON 문자열로 저장하고, 페이지를 다시 열 때는 저장된 값을 읽어와 복원했습니다. 데이터 저장과 불러오기 함수는 `storage.js`에 분리하고, 화면 렌더링과 이벤트 처리는 `app.js`에서 담당하도록 나누었습니다.

상단의 오늘의 한마디는 `async/await`와 `fetch()`를 사용해 비동기로 불러오도록 만들었습니다. 네트워크 오류나 API 호출 실패가 발생할 수 있으므로 `try/catch`를 사용했고, 실패하면 기본 문구가 표시되도록 처리했습니다. 모듈을 사용하기 위해 `script` 태그에는 `type="module"`을 적용했습니다.

## 📝 자기 평가

이번 과제에서는 JavaScript로 화면의 상태를 직접 관리하는 흐름을 연습할 수 있었습니다. 할 일 하나를 단순한 문자열이 아니라 `{ id, text, done }` 객체로 만들고, 전체 목록을 배열로 관리하니 추가, 완료, 삭제, 필터링 기능을 더 체계적으로 구현할 수 있었습니다. 화면에 보이는 목록은 결국 배열 상태를 기준으로 다시 그려지는 것이기 때문에, 상태 변경 후 `renderTodos()`를 다시 실행하는 흐름이 중요하다는 것을 알게 되었습니다.

할 일 추가 기능에서는 입력값을 그대로 사용하는 것이 아니라 `trim()`으로 공백을 제거하고 빈 값이면 추가되지 않도록 처리했습니다. 작은 부분이지만 사용자가 아무것도 입력하지 않았을 때도 앱이 이상하게 동작하지 않도록 막는 기본 검증이 필요하다는 것을 배웠습니다.

완료와 삭제 기능을 만들면서 배열 메서드의 역할도 더 확실히 이해했습니다. 새 할 일을 추가할 때는 `push()`를 사용했고, 삭제할 때는 지울 항목을 직접 제거하기보다 `filter()`로 남길 항목만 다시 구성했습니다. 완료 필터와 진행중 필터도 `filter()`를 사용해 현재 상태에 맞는 항목만 화면에 표시했습니다.

DOM을 다룰 때는 `innerHTML`로 한 번에 문자열을 넣는 방식보다 `createElement()`로 요소를 만들고 필요한 속성과 class를 붙이는 방식을 사용했습니다. 특히 각 할 일 항목의 id를 `dataset`에 저장해두니, 체크박스나 삭제 버튼을 눌렀을 때 어떤 항목을 처리해야 하는지 찾기 쉬웠습니다.

이벤트 위임도 이번 과제에서 인상적이었습니다. 처음에는 할 일 항목마다 이벤트를 붙여야 할 것 같았지만, 부모 `ul`에 이벤트 리스너를 한 번만 등록해도 동적으로 추가된 항목까지 처리할 수 있었습니다. 클릭된 요소가 체크박스인지 삭제 버튼인지 확인해서 동작을 나누는 방식이 효율적이라는 것을 배웠습니다.

`localStorage`를 사용하면서 브라우저에 데이터를 저장하고 다시 불러오는 흐름도 익혔습니다. 배열은 그대로 저장할 수 없기 때문에 `JSON.stringify()`로 문자열로 바꿔 저장하고, 다시 가져올 때는 `JSON.parse()`로 배열로 복원해야 한다는 점을 확인했습니다. 새로고침 후에도 할 일이 남아 있는 것을 보며 데이터 영속성의 개념을 이해할 수 있었습니다.

이번 실습에서는 `storage.js`와 `app.js`로 파일을 나누어 모듈도 사용했습니다. 데이터를 저장하고 불러오는 코드는 `storage.js`, 화면과 이벤트 처리는 `app.js`가 맡도록 분리하니 코드의 역할이 더 명확해졌습니다. 다만 ES 모듈은 파일을 더블클릭해서 열면 오류가 날 수 있고, Live Server나 `python -m http.server` 같은 로컬 서버로 실행해야 한다는 점도 다시 확인했습니다.

마지막으로 오늘의 한마디를 `fetch()`와 `async/await`로 불러오면서 비동기 처리도 복습했습니다. 외부 API는 인터넷 연결이나 서버 상태에 따라 실패할 수 있기 때문에 `try/catch`로 예외를 처리하고, 실패했을 때 기본 문구를 보여주는 fallback이 필요하다는 것을 배웠습니다. 단순히 성공하는 경우만 생각하는 것이 아니라 실패했을 때 화면이 어떻게 보여야 하는지도 함께 고려해야 한다는 점이 중요하게 느껴졌습니다.
