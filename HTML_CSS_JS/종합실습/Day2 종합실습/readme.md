## 01_반응형 상품 카드 갤러리

내 가방 속 소지품을 카드 형태로 보여주는 반응형 갤러리를 만들었습니다.

카드 목록은 CSS Grid의 `repeat(3, 1fr)`을 사용해 기본 3열로 배치했습니다. 화면 너비가 900px 이하일 때는 2열, 600px 이하일 때는 1열로 바뀌도록 Media Query를 적용했고, 작은 화면에서는 상단 메뉴를 숨겨 카드 내용에 집중할 수 있도록 구성했습니다.

상단바는 Flexbox로 정렬하고 로고, 메뉴, 다크모드 버튼의 위치를 나누었습니다. 색상은 `:root`에 CSS 변수로 정의한 뒤 `var()`로 사용했습니다. 다크모드에서는 `body.dark`의 변수 값만 교체해 배경, 카드, 글자와 버튼 색상이 함께 바뀌도록 만들었습니다.

카드에 마우스를 올리면 `transition`과 `transform`을 이용해 카드가 위로 떠오르고 그림자가 진해집니다. 페이지를 열면 히어로 문구와 카드가 순서대로 나타나도록 `@keyframes`와 `animation-delay`도 적용했습니다. 움직임에 민감한 사용자를 위해 `prefers-reduced-motion` 환경에서는 애니메이션이 비활성화되도록 처리했습니다.

### 주요 학습 내용

- CSS 변수와 `var()`를 활용한 색상 관리
- Grid를 이용한 3열, 2열, 1열 반응형 카드 배치
- Flexbox를 이용한 상단바 정렬
- 다크모드 테마 전환
- `transition`, `transform`, `@keyframes` 애니메이션
- `prefers-reduced-motion`을 이용한 접근성 대응

## 02_인터랙티브 할 일 관리 앱

할 일을 추가하고 완료하거나 삭제할 수 있는 할 일 관리 앱을 만들었습니다.

할 일 하나는 `{ id, text, done }` 형태의 객체로 만들고 전체 목록은 배열로 관리했습니다. 추가 버튼이나 Enter 키로 새 항목을 등록할 수 있으며, 빈 값은 추가되지 않도록 검사했습니다. 체크박스를 누르면 완료 상태와 취소선이 적용되고, 삭제 버튼을 누르면 해당 항목이 목록에서 제거됩니다.

전체, 진행중, 완료 버튼을 이용해 목록을 필터링하고 화면 아래에는 전체 개수와 완료 개수를 표시했습니다. 목록은 `createElement()`로 만들고 각 항목의 id는 `dataset`에 저장했습니다. 부모 `ul`에 이벤트 리스너를 한 번만 등록하는 이벤트 위임 방식으로 동적으로 추가된 항목까지 처리했습니다.

할 일 데이터는 localStorage에 JSON 문자열로 저장해 새로고침 후에도 유지되도록 했습니다. 저장과 불러오기는 `storage.js`, 화면 출력과 이벤트 처리는 `app.js`로 나누고 ES 모듈의 `import`와 `export`를 사용했습니다.

상단의 오늘의 한마디는 `fetch()`와 `async/await`로 외부 API에서 불러옵니다. 네트워크 요청이 실패할 경우에도 빈 화면이 나타나지 않도록 `try/catch`에서 기본 문구를 표시하도록 만들었습니다.

### 주요 학습 내용

- 객체 배열을 이용한 화면 상태 관리
- `push()`와 `filter()`를 이용한 추가 및 삭제
- DOM 생성과 `dataset` 활용
- 이벤트 위임을 이용한 동적 항목 처리
- localStorage와 JSON을 이용한 데이터 저장 및 복원
- `async/await`, `fetch()`, `try/catch`를 이용한 비동기 처리
- `import`, `export`, `type="module"`을 이용한 모듈 분리

## 실행 방법

할 일 관리 앱은 ES 모듈을 사용하므로 HTML 파일을 직접 더블클릭하지 않고 Live Server 또는 로컬 서버로 실행해야 합니다.

```bash
python3 -m http.server
```

서버를 실행한 뒤 브라우저에서 해당 폴더의 `index.html`로 접속하면 됩니다.

## 정리

Day2 종합실습에서는 화면의 배치와 색상을 만드는 것에서 더 나아가 화면 크기, 사용자 동작, 데이터 상태와 저장까지 함께 다뤘습니다. 첫 번째 과제를 통해 Grid와 Flexbox의 쓰임을 구분하고 반응형 페이지와 테마를 구성하는 방법을 익혔습니다. 두 번째 과제에서는 배열 상태가 바뀔 때 화면을 다시 그리는 흐름과 브라우저에 데이터를 보관하는 방법을 연습했습니다.

특히 CSS 속성이 애니메이션과 hover에서 겹치면 효과가 충돌할 수 있다는 점, ES 모듈은 로컬 서버 환경에서 실행해야 한다는 점, 외부 API는 실패할 가능성을 고려해 기본 동작을 준비해야 한다는 점을 확인할 수 있었습니다.
