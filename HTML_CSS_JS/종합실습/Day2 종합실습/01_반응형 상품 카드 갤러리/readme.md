# Day2 종합실습 - 01 반응형 상품 카드 갤러리

🗓️ 수행 날짜 : 2026-07-18

📚 수행 내용  
- CSS Grid와 Flexbox를 사용해 내 가방 속 소지품 카드 갤러리를 만들고, CSS 변수 기반 테마 관리, 다크모드 전환, hover 효과, 등장 애니메이션을 적용했습니다.

## Doing

이번 과제에서는 내 가방 속 소지품을 카드 갤러리 형태로 정리하면서 CSS Grid, Flexbox, CSS 변수, Media Query, transition, transform, keyframes를 함께 사용했습니다. 화면 크기에 따라 카드가 3열에서 2열, 1열로 자연스럽게 바뀌도록 구성하는 것이 핵심이었습니다.

먼저 `:root`에 배경색, 글자색, 카드 배경색, 포인트 색상, 테두리색, 그림자 색상 등 여러 색상 변수를 정의했습니다. 실제 스타일에서는 색상 값을 직접 반복해서 쓰지 않고 `var()`로 불러오도록 했습니다. 다크모드는 `body.dark`에서 같은 변수의 값만 바꾸는 방식으로 구현해, 전체 배경, 카드, 글자, 버튼 색상이 한 번에 전환되도록 만들었습니다.

상단바는 Flexbox를 사용해 정렬했습니다. 로고는 왼쪽에 두고, 메뉴와 다크모드 버튼은 오른쪽으로 정렬했습니다. 특히 메뉴 영역에는 `margin-left: auto`를 사용해 로고와 메뉴 사이의 공간이 자동으로 벌어지도록 했습니다.

소지품 카드 목록은 CSS Grid로 구성했습니다. 기본 화면에서는 `grid-template-columns: repeat(3, 1fr)`을 사용해 3열로 배치했고, 900px 이하에서는 2열, 600px 이하에서는 1열로 바뀌도록 Media Query를 적용했습니다. 600px 이하에서는 상단 메뉴도 숨겨 작은 화면에서 카드에 더 집중할 수 있도록 했습니다.

각 소지품 카드는 hover 시 `transform: translateY()`로 살짝 위로 떠오르고, 그림자가 더 진해지도록 만들었습니다. 이 변화가 갑자기 일어나지 않도록 `transition`을 적용해 부드럽게 보이도록 했습니다.

페이지가 처음 열릴 때는 히어로 문구와 소지품 카드가 아래에서 위로 순서대로 등장하도록 `@keyframes` 애니메이션을 정의했습니다. 카드마다 `animation-delay`를 다르게 주어 한 번에 모두 나타나는 것이 아니라 차례대로 등장하는 느낌을 주었습니다. 또한 접근성을 고려해 운영체제에서 동작 줄이기를 설정한 경우 애니메이션과 transition이 비활성화되도록 `prefers-reduced-motion`도 함께 적용했습니다.

## 📝 자기 평가

이번 과제에서는 CSS Grid와 Flexbox를 각각 어디에 쓰면 좋은지 조금 더 분명하게 이해할 수 있었습니다. 여러 개의 카드를 2차원으로 배치할 때는 Grid가 편했고, 상단바처럼 로고, 메뉴, 버튼을 한 줄로 정렬할 때는 Flexbox가 더 적합했습니다. 특히 `repeat(3, 1fr)`로 기본 3열을 만들고, Media Query에서 2열과 1열로 바꾸는 흐름을 직접 적용하면서 반응형 레이아웃의 기본 구조를 익힐 수 있었습니다.

CSS 변수도 유용하게 느껴졌습니다. 처음부터 색상을 `:root`에 변수로 정의해두니, 다크모드를 만들 때 전체 스타일을 하나씩 다시 수정하지 않아도 됐습니다. `body.dark`에서 변수 값만 바꿔도 배경색, 카드 색, 글자색, 버튼 색이 함께 바뀌는 것을 보면서 CSS 변수가 테마 관리에 적합하다는 것을 알게 되었습니다.

작업 중 hover가 제대로 동작하지 않는 문제가 있었습니다. 처음에는 카드 등장 애니메이션에서도 `transform`을 사용하고, hover 효과에서도 `transform: translateY()`를 사용했습니다. 그래서 애니메이션에서 사용한 transform 값과 hover에서 사용한 transform 값이 겹치면서 카드 hover가 기대한 대로 보이지 않았습니다. 이후 등장 애니메이션은 `translate` 속성으로 처리하고, hover는 `transform`을 사용하도록 분리해서 문제를 해결했습니다. 이 과정을 통해 같은 요소에 여러 효과를 줄 때 같은 CSS 속성을 동시에 쓰면 충돌이 생길 수 있다는 점을 배웠습니다.

히어로 영역의 등장 애니메이션도 처음에는 영역 전체가 한 번에 나타나는 방식이었습니다. 요구사항은 제목과 부제가 아래에서 순차적으로 등장하는 것이었기 때문에, `hero-label`, 제목, 설명 문장에 각각 `animation-delay`를 다르게 적용해 순서대로 나타나도록 수정했습니다. 이를 통해 애니메이션은 단순히 넣는 것보다 어떤 요소가 어떤 순서로 움직여야 하는지까지 생각해야 한다는 것을 알게 되었습니다.

또한 `prefers-reduced-motion`을 사용해 모션 최소화 환경에 대응하는 방법도 배웠습니다. 애니메이션은 화면을 더 자연스럽게 만들어주지만, 사용자에 따라 움직임이 불편할 수도 있기 때문에 접근성 관점에서 끌 수 있는 처리가 필요하다는 점이 인상적이었습니다. 개발자도구에서 reduced motion을 흉내 내고 새로고침해보며 애니메이션이 비활성화되는지도 확인할 수 있었습니다.

전체적으로 이번 실습은 단순히 예쁜 카드를 만드는 것보다, 레이아웃, 테마, 반응형, 상호작용, 접근성을 함께 고려해야 한다는 점을 배운 과제였습니다. 특히 문제가 생겼을 때 CSS 속성끼리 어떻게 영향을 주는지 확인하고 수정하는 과정이 가장 도움이 되었습니다.
