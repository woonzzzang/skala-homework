## Doing

회원가입 폼을 만들면서 여러 input 타입과 유효성 검증 속성을 함께 사용해봤습니다. 입력한 값은 `result.html`로 넘겨서 표 형태로 확인할 수 있게 했고, 폼에서 작성한 값이 실제로 어떻게 전달되는지 확인하는 데 중점을 두었습니다.

회원가입 페이지에서는 `form`, `input`, `select`, `textarea`, `button` 역할을 구분해서 사용했습니다. 이름, 아이디, 비밀번호, 이메일, 전화번호, 생년월일처럼 입력 방식이 다른 항목들을 각각 `text`, `password`, `email`, `tel`, `date` 타입으로 구성했고, 성별은 `radio`, 관심 분야는 `checkbox`, 지역은 `select`, 자기소개는 `textarea`를 사용했습니다.

브라우저 기본 유효성 검증도 함께 적용했습니다. 필수 항목에는 `required`를 사용했고, 아이디는 `pattern`으로 영문과 숫자 4~12자만 입력되도록 했습니다. 비밀번호에는 `minlength="8"`을 적용했으며, 전화번호는 `010-0000-0000` 형식에 맞도록 `pattern`을 지정했습니다. 생년월일에는 `min`과 `max`를 넣어 날짜 입력 범위를 제한했고, 자기소개에는 `maxlength`를 사용해 입력 가능한 글자 수를 제한했습니다.

입력 항목은 `fieldset`과 `legend`를 사용해 계정 정보, 개인 프로필 정보, 자기소개, 약관 동의 영역으로 그룹화했습니다. 이를 통해 화면 구조가 더 명확해지고, 사용자가 어떤 정보를 입력하는지 구분하기 쉬운 폼으로 정리했습니다.

CSS에서는 Box Model을 적용해 입력창, fieldset, 버튼, 테이블의 여백과 테두리를 정리했습니다. 또한 상단 헤더에 `position: fixed`와 `z-index`를 적용해 스크롤을 내려도 계속 상단에 고정되도록 만들었습니다. 입력창에는 `:focus` 스타일을 적용해 사용자가 현재 입력 중인 항목을 쉽게 알아볼 수 있게 했고, 버튼에는 `:hover` 스타일을 넣어 마우스를 올렸을 때 반응이 보이도록 했습니다.

폼 제출은 `method="get"` 방식으로 처리했습니다. 입력값은 URL 쿼리스트링으로 `result.html`에 전달되며, 결과 페이지에서는 JavaScript의 `URLSearchParams`를 사용해 전달된 값을 읽어와 table에 표시했습니다. 특히 관심 분야처럼 같은 `name`을 가진 checkbox 값은 여러 개가 전달될 수 있기 때문에 `getAll()`을 사용해 복수 값을 함께 표시하도록 구현했습니다.

기존에 만들었던 `signUp.html`과 `signUpResult.html`은 회원가입 화면과 완료 안내 페이지의 기본 흐름을 익히는 데 도움이 되었습니다. 이번 종합실습에서는 그 흐름을 확장해 입력값 검증, 결과 페이지 이동, table 출력, 복수 값 처리, 고정 헤더까지 함께 적용했습니다.

## 📝 자기 평가

이번 과제에서는 HTML Form이 단순히 화면에 입력창을 보여주는 요소가 아니라, 입력값을 모으고 검증한 뒤 다른 페이지로 전달하는 흐름까지 담당한다는 것을 배웠습니다. 특히 `name` 속성이 있어야 폼 전송 시 값이 전달되고, 같은 `name`을 가진 checkbox는 복수 값으로 넘어간다는 점을 직접 확인할 수 있었습니다.

이전에는 `signUp.html`에서 회원가입 화면을 만들고 `signUpResult.html`에서 축하 메시지를 보여주는 정도로 구현했지만, 이번에는 그 흐름을 확장해 실제 입력값을 결과 페이지에 표로 출력하도록 만들었습니다. 이를 통해 `form action`, `method=get`, URL 쿼리스트링, 결과 페이지에서 값 읽기까지의 연결 과정을 이해할 수 있었습니다.

또한 `required`, `pattern`, `minlength`, `maxlength`, `min`, `max` 같은 HTML 속성만으로도 JavaScript 없이 기본적인 유효성 검증이 가능하다는 점이 인상적이었습니다. 빈 필수값을 제출하거나 아이디 패턴을 어기거나 비밀번호 길이가 짧을 때 브라우저가 자동으로 제출을 막아주는 것을 보며, HTML 자체가 제공하는 기능을 먼저 잘 활용하는 것이 중요하다고 느꼈습니다.

CSS에서는 `position: fixed`와 `z-index`를 사용해 고정 헤더를 만들면서, 고정된 요소가 본문을 가릴 수 있기 때문에 body의 위쪽 여백도 함께 고려해야 한다는 점을 배웠습니다. 또한 `:focus`와 `:hover`를 적용하면서 사용자의 행동에 따라 화면이 반응하는 작은 스타일이 사용성을 높이는 데 도움이 된다는 것을 알게 되었습니다.

이번 실습을 통해 회원가입 폼, 유효성 검증, 결과 페이지, CSS 배치와 상태 스타일을 한 번에 연결해볼 수 있었습니다. 다음에는 결과 페이지의 값을 JavaScript로 더 보기 좋게 가공하거나, 필수 입력 값 입력 시 어떤 효과들을 적용해보고 싶습니다.
