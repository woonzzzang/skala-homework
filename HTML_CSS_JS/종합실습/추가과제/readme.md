# 정다운 아카이브

🗓️ 수행 날짜 : 2026-07-18 ~ 2026-07-19<br>
👤 작성자 : 4기 광주 3반 정다운<br>
🔗 배포 주소 : [https://woonzzzang.github.io/daun-archive/](https://woonzzzang.github.io/daun-archive/)<br>

지금까지 HTML, CSS, JavaScript 수업에서 만든 실습과 과제를 하나의 개인 아카이브 사이트로 묶었습니다. 단순히 결과물을 모아두는 데 그치지 않고, 제 소개와 여행 기록, 가방 속 물품, 할 일, 수업 시간표와 식단을 한 곳에서 확인할 수 있도록 구성했습니다.

## 폴더 구성

```text
추가과제
├── index.html                 # 메인 Hub 페이지
├── myProfile.html             # 자기소개
├── signUp.html                # 회원가입
├── result.html                # 회원가입 결과
├── login.html                 # 로그인
├── trip.html                  # 여행 기록
├── bag.html                   # 내 가방 속 소지품
├── todo.html                  # 할 일 관리
├── myClass.html               # 기존 수업 시간표 페이지
├── myHoliday.html             # 기존 휴일 계획 페이지
├── myTrip.html                # 기존 여행 앨범 페이지
├── signUpResult.html          # 기존 회원가입 결과 페이지
├── css/                       # 페이지별 스타일
├── script/                    # JavaScript 파일
├── data/                      # 시간표·식단 CSV 파일
└── media/                     # 사진, GIF, 오디오, 영상 파일
```

## Doing

### 1. 메인 Hub 페이지

`index.html`을 아카이브의 시작 페이지로 구성했습니다. 바로가기 메뉴에서 소개, 회원가입, 여행 기록, 내 가방, 할 일 관리 페이지로 이동할 수 있습니다.

메인 오른쪽에는 이번 주 시간표와 실시간 정보를 배치했습니다. 시간표는 `data/schedule.csv`를 읽어 날짜별 수업과 강사를 표시하고, 이전 주와 다음 주 버튼으로 주차를 이동할 수 있습니다. 시간표 아래의 점심시간 버튼을 누르면 `data/lunch-menu.csv`에서 읽은 해당 주 중식 메뉴가 팝업으로 표시됩니다.

메인에 있는 TO DO 메모는 할 일 관리 페이지와 같은 `localStorage` 데이터를 사용합니다. 진행 중인 할 일만 보여주며, 체크하면 완료 처리되어 메모에서 사라집니다. `+` 버튼으로 메인에서도 새로운 할 일을 추가할 수 있습니다.

업다운 게임과 성적 계산기는 기존의 `prompt()`와 `alert()` 방식 대신 페이지 안에서 사용할 수 있는 팝업으로 변경했습니다. 업다운 게임은 숫자 입력, Up/Down 힌트, 시도 횟수를 표시하고, 성적 계산기는 HTML·CSS·JavaScript 점수를 입력해 총점, 평균, 합격 여부를 보여줍니다.

### 2. 자기소개 페이지

`myProfile.html`에 프로필 사진과 함께 제 좌우명, 좋아하는 음식, MBTI, 좋아하는 캐릭터, 여행지, 야구팀, 운전면허 정보를 정리했습니다. 좋아하는 캐릭터인 하치와레 옆에는 GIF 이미지를 배치해 정적인 소개 페이지와 다른 느낌을 주었습니다.

### 3. 회원가입과 로그인

`signUp.html`에서는 이름, 아이디, 비밀번호, 이메일, 전화번호, 생년월일, 성별, 관심 분야, 지역, 자기소개와 약관 동의를 입력할 수 있습니다. `required`, `pattern`, `minlength`, `maxlength` 등의 HTML 속성을 사용해 브라우저 기본 유효성 검증을 적용했습니다.

회원가입 결과는 `result.html`에서 입력값을 표로 확인할 수 있도록 구성했습니다. 로그인 페이지는 `auth.js`에서 로그인 상태를 `localStorage`에 저장하고, 로그인 전에는 회원가입·로그인 메뉴를, 로그인 후에는 로그아웃 메뉴를 표시하도록 만들었습니다.

### 4. 여행 기록

`trip.html`에는 도쿄 디즈니씨, 오사카 글리코상, 후쿠오카 이치란 라멘 본점 사진을 명소 갤러리로 배치했습니다. 오사카 일루미네이션 영상은 `video` 요소와 `controls`, `poster`, `source`를 사용해 재생할 수 있도록 했습니다.

여행 기록 페이지는 명소, 먹거리, 영상, 팁으로 이동할 수 있는 내부 내비게이션을 제공합니다. 글로벌 헤더는 화면 상단에 고정하고, 여행 페이지의 내부 메뉴는 스크롤 위치에 맞춰 함께 확인할 수 있도록 구성했습니다. CSS에서는 이미지와 gradient를 겹친 히어로 배경, 카드 hover 확대 효과, 반응형 2단·1단 배치를 적용했습니다.

### 5. 내 가방

`bag.html`의 소지품 카드는 HTML에 직접 작성하지 않고 `script/bag.js`의 `myBag` 객체 배열을 기준으로 생성합니다. 각 객체에는 물품명, 이모지, 수량, 분류, 설명을 저장했습니다.

`showMyBag()` 함수가 `myBag`을 순회하면서 `createElement()`로 카드와 내용을 만들고 `#bag-list`에 추가합니다. 물품을 수정하거나 추가할 때는 HTML을 수정하지 않고 `myBag` 배열만 변경하면 카드와 총 물품 종류가 함께 바뀝니다.

### 6. 할 일 관리

`todo.html`에서는 할 일을 추가, 완료, 삭제하고 전체·진행중·완료 상태로 필터링할 수 있습니다. 하나의 할 일은 `{ id, text, done }` 객체로 관리하고 전체 목록은 배열에 저장했습니다.

`todoApp.js`는 DOM 생성과 이벤트 처리를 담당하고, `todoStorage.js`는 `localStorage` 저장과 복원을 담당합니다. 배열 변경 후에는 다시 렌더링해 화면을 갱신하고, 부모 `ul`에 이벤트 리스너를 한 번만 등록하는 이벤트 위임 방식을 사용했습니다.

상단의 오늘의 한마디는 `fetch()`와 `async/await`로 외부 API에서 불러옵니다. 인터넷 연결이나 API 호출에 문제가 생기면 `try/catch`에서 기본 문구를 표시하도록 처리했습니다.

## 공통 스타일과 기능

- 모든 주요 페이지에 정다운 아카이브 글로벌 헤더 적용
- 메뉴, 다크모드 버튼, TOP 버튼 위치 통일
- 다크모드 상태를 `localStorage`에 저장해 페이지를 이동해도 유지
- 카드, 입력창, 버튼, 팝업의 모서리 반경을 `8px`로 통일
- CSS Grid와 Flexbox를 이용한 레이아웃 구성
- 화면 너비에 따른 반응형 배치
- 이미지의 `alt`와 영상의 `controls`, `poster`, `source` 적용
- 팝업 바깥 영역 클릭과 `Escape` 키를 이용한 닫기
- 모바일 화면에서 메뉴를 숨기고 콘텐츠 중심으로 표시

## 결과

메인 Hub에서 각 메뉴로 이동할 수 있고, 여행 기록·내 가방·할 일 관리·회원가입 기능을 각각 독립적으로 확인할 수 있습니다. 메인에서는 시간표, 주간 중식 메뉴, 실시간 날씨와 진행 중인 할 일을 한 화면에서 확인할 수 있도록 구성했습니다.

## 실행 방법

ES 모듈과 CSV 파일을 사용하므로 HTML 파일을 직접 더블클릭하지 않고 Live Server 또는 로컬 서버에서 실행해야 합니다.

```bash
python3 -m http.server
```

서버를 실행한 뒤 브라우저에서 `index.html`을 열면 메인 Hub에서 각 페이지로 이동할 수 있습니다. 외부 날씨 API와 오늘의 한마디는 인터넷 연결이 필요하고, 연결되지 않을 때는 일부 기능이 기본 문구로 표시됩니다.

## 배운 점과 자기 평가

처음에는 수업에서 만든 페이지를 각각 따로 열어 확인했지만, 이번 과제를 진행하면서 여러 결과물을 하나의 사이트 구조로 연결하는 방법을 생각하게 되었습니다. 페이지마다 헤더와 메뉴를 따로 작성하면 조금씩 모양이 달라질 수 있어서 글로벌 헤더와 공통 스타일을 만들어 메뉴와 다크모드 버튼의 위치를 맞췄습니다.

특히 같은 데이터를 여러 화면에서 사용하려면 데이터를 HTML에 반복해서 적기보다 JavaScript 객체나 배열로 관리하는 것이 편하다는 것을 배웠습니다. `bag.js`의 `myBag` 배열에서 카드를 만들도록 수정하면서 데이터와 화면을 분리하는 방식의 장점을 직접 확인했습니다. 메인 TO DO 메모와 할 일 관리 페이지가 같은 `localStorage`를 공유하도록 만든 것도 같은 데이터를 여러 화면에서 활용한 예입니다.

작업 중에는 카드 hover와 등장 애니메이션에서 같은 `transform` 속성이 겹쳐 효과가 제대로 보이지 않는 문제가 있었습니다. 애니메이션에는 `translate`를 사용하고 hover에는 `transform`을 사용하도록 나누면서 해결했고, 하나의 요소에 여러 효과를 적용할 때 CSS 속성 충돌을 확인해야 한다는 점을 알게 되었습니다.

시간표와 식단표는 CSV 파일을 fetch로 읽은 뒤 필요한 행을 골라 화면에 표시했습니다. 날짜를 기준으로 주차를 나누고, 점심시간 버튼을 눌렀을 때만 중식 메뉴 팝업을 열도록 구성하면서 단순한 정적 HTML과 데이터를 불러와 화면을 만드는 방식의 차이도 경험했습니다.

또한 `prompt()`와 `alert()`로 동작하던 업다운 게임과 성적 계산기를 직접 만든 팝업으로 바꾸면서 사용자 입장에서 입력 과정과 결과를 한 화면에서 확인하는 것이 더 편하다는 점을 느꼈습니다. 기능을 구현하는 것뿐 아니라 사용하기 편한 화면과 예외 상황까지 함께 생각해야 한다는 것을 배운 작업이었습니다.
